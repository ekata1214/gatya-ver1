## ここでいう「画面構成」とは

この演出は **1 ページの `#stage` の中で最後まで完結** します。別 HTML への遷移（シーンカット）はありません。

**画面構成** = その1フレームの中で

1. **どの DOM 要素が重なっているか**（レイヤー）
2. **各レイヤーの絵は何が描いているか**（描画方式）
3. **時間とともに誰が値を更新するか**（駆動）

の3点を整理したものです。

フェーズごとに別アニメファイルがあるわけではなく、

1. **ステージ1枚**（1080∶1920 固定）
2. **レイヤー枠**（video / canvas / div — 最初から DOM に存在）
3. **フェーズごとの表示 ON/OFF と中身**
4. **GSAP タイムライン + コールバック**（マスタークロック）

で全部つながっています。

## 1フレームのイメージ

動画編集や Photoshop に慣れている人向けのたとえ：

> **各瞬間 = レイヤーを全部重ね合わせた状態を、1枚スクリーンショットしたもの**

```
┌──────────────────────── #stage（1080×1920） ────────────────────────┐
│                                                                        │
│   [UI: before/after, リプレイ]                          z: 1000 付近  │
│                                                                        │
│   ┌──────────────────────────────────────────────────────────────┐  │
│   │ 手前: ホワイトアウト → 暗転 → カウントダウン → SSR → 六角筒…  │  │
│   │        （同時に全部見えるわけではない）                        │  │
│   └──────────────────────────────────────────────────────────────┘  │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

ブラウザはこの重ね合わせを **秒間約60回** 更新しているだけです。フィルムのコマ送り用の画像列はありません。

## 混同しやすい2つの概念

### 1. レイヤー（合成枠）

**z-index で順序が決まった DOM 要素** のこと。

- `#fire-bg`
- `#cards-canvas`
- `#ssr-layer`
- など

レイヤーは **演出開始前から DOM に存在** します。多くは `opacity: 0` で非表示のまま待機し、自分のフェーズで表示されます。

### 2. 描画方式（中身を誰が描くか）

そのレイヤーの **ピクセルを誰が生成するか**：

| 描画方式 | 例 |
|----------|-----|
| HTML5 `<video>` | `#fire-bg`, `#ink-test`, `#white-ssr` |
| WebGL（Three.js） | `#cards-canvas`, `#countdown-canvas`, `#ssr-card-canvas` |
| DOM + CSS 3D | `#ssr-card-wrap` の transform |
| CSS の単色 div | `#darken`, `#whiteout` |
| CSS filter | `#cards-canvas` への `blur()` |

**駆動（動かす側）** はだいたい **GSAP**。動画は **`play()` / `currentTime`**。Three.js は **値が決まったあと描くだけ** です。

## 1コマ生成の流れ（基本モデル）

動きのある部分は、毎フレーム次の順番です：

```
時間（GSAP または動画の再生位置）
    → 状態を更新（数値・transform・opacity・動画時刻）
    → 各レイヤーがピクセルを生成（render / レイアウト / デコード）
    → ブラウザが下から上へ合成
    → 画面に1コマ表示
```

| 対象 | 状態を更新 | 絵の出力 |
|------|-----------|----------|
| 六角筒 | GSAP → `anim`, `spin` など | Three.js `#cards-canvas` |
| カウントダウン | GSAP → メッシュの scale/rotation | Three.js `#countdown-canvas` |
| SSR カード | GSAP → `#ssr-card-wrap` | DOM 配置 + 子 canvas |
| 炎 / 墨 / 白 SSR | `play()`, `currentTime` | 動画デコーダ |
| 暗転 / ホワイトアウト | GSAP → `opacity` | CSS 背景 |

**Three.js は演出の時計ではない。** 時計が「3D の状態を変えた」ときに **描き直す** だけです。

---

## Three.js は3つ（1シーンではない）

よくある誤解：**Three.js 1つ = 演出全体** ではありません。

| キャンバス | 中身 | 動きの駆動 |
|-----------|------|-----------|
| `#cards-canvas` | 六角筒（カード6枚） | GSAP →  plain object → `applyState()` |
| `#countdown-canvas` | 3D 文字 5→1→LAST | GSAP → メッシュ属性 |
| `#ssr-card-canvas` | 平面的なカード1枚 | 親 DOM の transform（GSAP） |

**シーングラフは共有しません。** フェーズが変わるときは、オブジェクトを移すのではなく **キャンバスの表示/非表示を切り替える** ことが多いです。
